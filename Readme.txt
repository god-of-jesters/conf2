ver: 1.0.0

Этап 1 — Минимальный прототип + конфигурация
- Реализован CLI для приема пользовательских настроек.
- Параметры: package, repo/url, mode, version, filter.
- Все параметры выводятся в формате ключ-значение.
- Реализована базовая обработка ошибок (формат пакета, пустые значения).
- GUI (tkinter) добавлен только как вывод для демонстрации параметров.

Этап 2 — Сбор данных
- Добавлено получение POM-файла Maven из удалённого или локального репозитория.
- Реализовано парсирование прямых зависимостей из POM (без parent / dependencyManagement).
- Добавлена фильтрация зависимостей по подстроке.
- Реализована обработка ошибок загрузки/чтения/парсинга.
- Вывод всех прямых зависимостей на экран.

Этап 3 — Основные операции (граф)
- Реализован построитель графа зависимостей.
- Итеративный DFS (без рекурсии).
- Полная поддержка транзитивных зависимостей.
- Обработка циклов (поиск и вывод).
- Игнорирование узлов по подстроке (--filter).
- Добавлен тестовый режим: загрузка графа из файла вида:
    A: B C  
    B: D
- Демонстрация работы на разных тестовых графах.

Этап 4 — Дополнительные операции
- Добавлен "порядок загрузки зависимостей" (load order):
  постфиксный DFS: сначала зависимости, затем узел.
- Поддержка флага --show-load-order.
- Сравнение с Maven: различия из-за отсутствия parent, scopes, exclusions.
- Отмечены причины расхождений: Maven скрывает конфликтующие версии,
  применяет dependencyManagement, фильтрует по scope — прототип нет.

Этап 5 — Визуализация
- Реализовано преобразование графа в язык диаграмм D2.
- Добавлен вывод генерации D2 прямо в консоль (--d2).
- Простая визуализация графа через tkinter (узлы по кругу + рёбра).
- Продемонстрированы графы для трёх разных пакетов/тестовых графов.
- Сравнение с визуализацией Maven:
  Maven показывает "разрешённое дерево", наш прототип — "сырой граф".

Основные функции (краткое описание)
- parse_args() — парсинг аргументов командной строки.
- build_pom_path() — формирование пути/URL для POM файла Maven.
- fetch_pom_remote/local() — загрузка POM с сети или из файловой системы.
- parse_pom_direct_deps() — извлечение прямых зависимостей из POM.
- load_test_repo() — загрузка тестового графа зависимостей из файла.
- build_transitive_graph() — построение графа:
    * итеративный DFS
    * транзитивные зависимости
    * фильтрация
    * обработка циклов
    * формирование порядка загрузки
- graph_to_d2() — генерация D2 текста.
- show_graph_tk() — визуальное отображение графа в окне tkinter.
- main() — объединяет все этапы и формирует итоговый CLI-интерфейс.
